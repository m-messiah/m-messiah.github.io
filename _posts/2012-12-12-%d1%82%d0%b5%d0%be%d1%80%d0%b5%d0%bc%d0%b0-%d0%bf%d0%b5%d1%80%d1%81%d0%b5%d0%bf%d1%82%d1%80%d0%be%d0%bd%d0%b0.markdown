<h1>Теорема Перцептрона<&#47;h1>
<address>Если есть множество точек на плоскости, как построить разделяющее множество?<&#47;address><address><a id="more"></a><a id="more-225"></a><&#47;address>Можно выделить две выпуклые оболочки, найти ближайшие точки и между ними по центру провести прямую. Под каким углом?
Ну ок, в двумерном клёво, а в n-мерном - огромные вычислительные сложности на построение выпуклой оболочки.
Алгоритм общего перебора.
$latex x_1..x_k$ - промежуточный класс

$latex y_1..y_k$ - правильный класс.
Надо найти вектор $latex (w_1..w_n)$ и порог, тогда $latex g(x)=\sum{w_i x_i}$
Решающее правило: $latex f(x)= sign{g(x)}$
<h2>Алгоритм:<&#47;h2>
<ol>
	<li>Пробегаем 1..k
$latex \sigma (x_k) = y_k$ ???
<ul>
	<li>Да - ничего<&#47;li>
	<li>Нет - $latex w:=w+y_k*x_k$
$latex \theta := \theta + y_k$<&#47;li>
<&#47;ul>
<&#47;li>
	<li>На шаг 1<&#47;li>
	<li>Нашли конец - хорошо. Были ошибки - плохо.<&#47;li>
<&#47;ol>
<h2>Формулировка теоремы.<&#47;h2>
Если классы -1,1 линейноразделимы, т.е. [latex]\exists W^* \epsilon \Re ^n ; \theta^* \epsilon \Re ^n: \forall K: (\sum{w_i^* x_i^k} + \theta^*)y_k>0=>[&#47;latex] алгоритм сходится за конечное время.
